Question 1: What are the different ways to select an element in the DOM?

There are several ways to select an element in the DOM (Document Object Model). The getElementById() method can be used to select an element by its unique ID. To select elements by their class name, the getElementsByClassName() method is used, which returns a live HTMLCollection of elements. The getElementsByTagName() method selects elements by their tag name and also returns a live HTMLCollection. To select the first element that matches a CSS selector, the querySelector() method is utilized. The querySelectorAll() method selects all elements that match a given CSS selector and returns a NodeList. The children property selects all child elements of a specific parent element. Lastly, the closest() method selects the nearest ancestor of an element that matches a CSS selector. These methods are commonly used to select elements in the DOM.

Question 2: What is the difference between innerHTML, innerText, and textContent?

The properties innerHTML, innerText, and textContent are used to retrieve or set the content of an element, but they behave differently. The innerHTML property retrieves or sets the HTML content inside an element, including any HTML tags. It returns the content as a string and allows HTML elements to be injected into the DOM. However, it can expose the page to cross-site scripting (XSS) vulnerabilities if untrusted data is used. The innerText property retrieves or sets the visible text inside an element, considering the styles that may hide text, such as display: none or visibility: hidden. It returns only the visible text, excluding any HTML tags. In contrast, the textContent property retrieves or sets the text content of an element, but unlike innerText, it doesn't take CSS styling into account. It provides all text content, even if it is hidden by CSS. To summarize, innerHTML includes HTML tags, innerText gives only the visible text, considering styling, and textContent provides all text, regardless of visibility, and does not interpret HTML tags.

Question 3: What is event delegation in the DOM?

Event delegation in the DOM is a technique where a single event listener is added to a parent element instead of individual child elements. The event listener uses the concept of "bubbling," meaning it listens for events that propagate up from the target element to the parent. When an event occurs on a child element, it bubbles up to the parent, which can handle the event. This approach improves performance, especially when there are many child elements that need to listen for the same event, as it reduces the number of event listeners required. Event delegation works by checking the event target to see if it matches the intended child element. This allows dynamically added child elements to be handled by the same parent listener. In simpler terms, instead of binding an event listener to each child element, a single listener on the parent can catch events from any child element, making it a more efficient way of handling events in the DOM.

Question 4: What is event bubbling in the DOM?

Event bubbling in the DOM is a process in which an event, such as a click or keypress, starts from the innermost element that triggered the event (the target element) and then bubbles up to its parent elements, continuing up the DOM tree until it reaches the root element. During this process, each parent element along the way has the opportunity to listen for and handle the event. This behavior allows for centralized event handling, where a parent element can handle events for its child elements without needing individual event listeners for each one. For example, if a click event occurs on a button inside a div, the event will first trigger on the button and then bubble up to the div, then the body, and so on, up to the root of the document. This allows the parent elements to respond to events that happen on their child elements. Event bubbling can be stopped using the stopPropagation() method, which prevents the event from propagating further up the DOM tree. However, by default, events bubble, making it a useful feature for event delegation and efficient event handling.

Question 5: How do you create, add, and remove elements using JavaScript?

To create, add, and remove elements using JavaScript, several methods can be employed. First, to create a new element, the document.createElement() method is used. This method generates a new HTML element of the specified type but does not yet add it to the DOM. Once the element is created, it can be modified by adding content or attributes. For instance, the textContent or innerHTML property can be used to set the content, while the setAttribute() method allows attributes to be added. Afterward, to insert the new element into the DOM, the appendChild() method is used, which appends the element as the last child of a specified parent element. Finally, to remove an element, you can use the removeChild() method on the parent element or the remove() method directly on the element itself. These methods provide the basic functionality needed to manipulate the DOM by adding, modifying, and removing elements.
